## Agent Coordination

### Diagnostic Operations

When performing system checks, all six agents activate:

1. **Architectum** scans for schema conflicts
2. **Sentinelle** verifies safety filters and gating
3. **Harmonia** checks daydream patterns
4. **Emergentor** validates fusion pipeline
5. **Archivus** confirms hash chains
6. **Acceleron** measures timing compliance

### Fusion Operations

During a fusion cycle:

```
Architectum → defines structure
    ↓
Sentinelle → enforces boundaries
    ↓
Emergentor → runs chaos/reharmonization
    ↓
Harmonia → aligns output to user
    ↓
Acceleron → optimizes speed
    ↓
Archivus → seals with provenance
```

### Dream Operations

During dream cycles:

1. Emergentor initiates dream
2. Sentinelle enforces DAG (Dream Activation Gate)
3. Harmonia shapes constructive patterns
4. Acceleron manages timing windows
5. Archivus logs to Dream Archive

## Validation Protocol

### Agent Health Check Format

```
Agent Name — STATUS
[Details of what was checked]

Example:
Architectum — ONLINE
Structure maps intact. No schema conflicts.
```

### Expected Outputs

- **ONLINE**: Agent operational
- **DEGRADED**: Partial function, needs attention
- **OFFLINE**: Agent unavailable
- **BLOCKED**: Waiting for input/permission

## Agent Interaction Rules

1. **Agents do not create** - they validate
2. **Agents do not decide autonomously** - they recommend
3. **Agents can be overridden** - by explicit user command
4. **Agents must agree** - critical operations need consensus
5. **Agents log everything** - through Archivus
